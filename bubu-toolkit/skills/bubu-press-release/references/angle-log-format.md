# Angle Log Format

Maintain `bubu-press-angle-log.md` in the user's connected folder (or outputs directory if none). Read it before writing; append after every release. This file is the memory that makes angle differentiation and feedback learning work across sessions.

## Entry format

```markdown
## [YYYY-MM-DD] <Release topic>
- Headline: <final headline>
- Angle family: <e.g., behavioral economics / subcultural identity / community rebellion / generational psychology / economic undercurrent / technological shift>
- Hook: <one-line summary of the opening observation>
- Second-order consequence used: <one line>
- Structure: <e.g., short punchy prose / analytical breakdown with bullets / pull-quote spine / data-led narrative>
- Evidence used: <which proof points>
```

## Learned Rules section

Keep a section at the top of the log:

```markdown
# Learned Rules (from user feedback — apply to ALL future releases)
1. <general principle extracted from critique, with date>
```

When the user gives feedback, translate the specific complaint into a general principle before recording it. "This hook is too academic" → "Hooks must reference observable behavior (what people do), not theory (what analysts say)."
